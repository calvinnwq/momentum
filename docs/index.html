01Overview

AI-native workflow backbone.

Momentum is a skeleton for creating AI-native repo workflows: reusable workflows, ordered steps, pluggable executors, source adapters, durable evidence, gates, recovery actions, and operator-readable surfaces. The Momentum-native coding workflow is the first shipped recipe; the original goal-first command lane is retired, and its stored goal rows and artifacts remain readable compatibility data.

Choose an entry point

The fastest way to understand Momentum today is to preview the shipped coding workflow, then inspect the durable run surfaces it writes. Custom workflows, steps, executors, and adapters use the same backbone as those surfaces grow.

Shipped workflow Preview first, then start a durable coding run. Use this path to see workflow definitions, steps, approvals, monitor/watch events, and tail-step recovery in one run. End-to-end smoke Start, schedule, and inspect one durable workflow run. Use this path for a copy-paste smoke that composes workflow run start, a bounded daemon cycle, and the read-only inspectors. Agent operation Install the public Momentum skill. Use this path when an agent host needs CLI discovery, workflow run guidance, gate handling, and evidence summaries. 
# build the CLI from source
pnpm install
pnpm build

# check local runtime readiness
node dist/index.js doctor --json

# preview a Momentum-native coding workflow without writing state
node dist/index.js workflow run preview-coding \
  --run-id <run-id> \
  --repo <repo> \
  --objective "Make the requested change" \
  --approval-boundary through-no-mistakes \
  --json

# or start a durable workflow run and inspect it
node dist/index.js workflow run start --run-id demo-1 --repo /path/to/repo --objective "Make the requested change" --json
node dist/index.js workflow status demo-1 --json
node dist/index.js workflow run logs demo-1 --json
node dist/index.js workflow handoff demo-1 --json

Process-backed workflow execution is supported on Linux and macOS. Native Windows is not a supported or CI-proven host; platform-portable parsing remains available, while executable steps refuse with unsupported_platform before a supervised command is spawned.

For a copy-paste workflow smoke test, use the end-to-end walkthrough. For the native workflow contract, start with workflow commands.

What Momentum owns

Workflow backbone Definitions, steps, runs, approvals, gates, monitors, watch ticks, and logs. The built-in coding workflow is one recipe on top of the same durable runtime intended for user-defined workflows. Evidence and adapters External state is read, mirrored, and reconciled through local records. Source adapters, evidence rows, and update intents connect workflow state to systems like trackers without making every read a write. Daemon and recovery Scheduling, leases, gates, and manual recovery are runtime state. The daemon and monitor surfaces decide what can advance, what needs approval, and what must be recovered before retry. Executors and apply Bounded work lives below steps; side effects stay explicit. Executor families such as goal-loop, one-shot, script, no-mistakes, and external-apply produce normalized evidence the workflow can resume from.

Runtime map

Momentum separates the workflow definition, the durable runtime state, the executor doing bounded work, and the adapters or side effects that require evidence-backed reconciliation.

Workflow authors Definitions and step recipes built-in coding today; custom workflows over time
Runtime backbone SQLite plus artifacts runs, steps, leases, gates, events, evidence
Executors goal-loop, one-shot, script, no-mistakes bounded work with normalized results and logs
Adapters and systems Git, GitHub, Linear, future sources read first, mutate only through approved apply paths

Operator surfaces

These are the everyday commands for observing a run and choosing the next safe action. Read-only commands are designed for humans, agents, GUI clients, and scheduled supervisors.

momentum workflow status <run-id> --jsonInspect one durable run: steps, approvals, leases, gates, and recovery flags. momentum workflow run watch --once --jsonPoll one native workflow and receive the next recommended action. momentum workflow handoff <run-id> --jsonEmit a compact restart packet with the same durable facts. momentum openclaw supervise <run-id> --once --jsonOpenClaw supervise wraps watch output for host delivery and local auto-actions.
