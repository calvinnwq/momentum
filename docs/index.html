01Overview

Durable repo work, without terminal archaeology.

Momentum turns autonomous coding work into explicit workflow state: durable runs, resumable steps, evidence records, gates, recovery actions, and operator-readable surfaces.

Plan Run Record Recover Reconcile

Start with a disposable run

The fastest way to understand Momentum is to run a tiny goal, inspect the state, then read the handoff and logs. The CLI is pre-release, so local source install is the supported path. 

# build the CLI from source
pnpm install
pnpm build

# check local runtime readiness
node dist/index.js doctor --json

# run a foreground goal from a Markdown spec
node dist/index.js goal start ./goal.md --foreground --json
node dist/index.js status --json
node dist/index.js handoff <goal-id> --json

For a copy-paste smoke test, use the end-to-end walkthrough.

What Momentum owns

Workflow runtime Runs, steps, approvals, gates, monitors, watch ticks, and logs. The native coding workflow is represented as durable rows and JSON envelopes. Evidence Artifacts and proof become queryable records. Evidence links goals, source items, external intents, workflow state, and recovery. Recovery Manual recovery is state, not vibes in a transcript. Stale leases, failed runners, external-tail reconcile, and no-mistakes evidence lanes are explicit. External apply Tracker writes are policy-gated and audited before mutation. Linear updates are intents first, then approved external apply or workflow tail reconciliation.

Runtime map

Momentum separates the user command, durable workflow state, the executor doing bounded work, and the side effects that require evidence-backed reconciliation.

Operator CLI or host wrapper start, status, monitor, approve, recover
Workflow state SQLite plus artifacts runs, steps, gates, events, evidence
Executors goal-loop, one-shot, no-mistakes bounded work with normalized results
External systems Git, GitHub, Linear read first, mutate only through approved paths

Operator surfaces

These are the everyday commands for observing a run and choosing the next safe action. Read-only commands are designed for humans, agents, GUI clients, and scheduled supervisors.

momentum statusInspect active goal state and current recovery flags. momentum workflow run watch --once --jsonPoll one native workflow and receive the next recommended action. momentum handoff <goal-id>Emit a compact restart packet with the same durable facts. momentum openclaw supervise <run-id> --once --jsonOpenClaw supervise wraps watch output for host delivery and local auto-actions.
